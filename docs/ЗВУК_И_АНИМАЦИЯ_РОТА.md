# Звук и анимация рта робота — полная инструкция

## 1. Что это и зачем

**Цель:** Когда робот издаёт любой звук (голос, сигнал, музыка), он должен:
1. Воспроизводиться через **звуковую карту** робота (динамики/наушники)
2. Одновременно на **OLED-экране** показываться анимация рта (открытие/закрытие по громкости)

**Раньше:** Звук шёл через SoX (`play`), анимация не была связана со звуком.

**Теперь:** Звук и анимация идут через один ROS-сервис — запускаются одновременно.

---

## 2. Схема: как всё связано

```
┌─────────────────────────────────────────────────────────────────────────┐
│ КТО ИЗДАЁТ ЗВУК                                                          │
├─────────────────────────────────────────────────────────────────────────┤
│ • app_node (при старте): voice_play.play('running')                     │
│ • oled_display (низкая батарея): voice_play.play('warnning')             │
│ • Ваш код: voice_play.play('running')                                    │
│ • Вручную: rosservice call /oled_mouth/play_audio "data: 'running'"      │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ voice_play.play() — ВЫБОР ПУТИ                                          │
├─────────────────────────────────────────────────────────────────────────┤
│ Если ROS запущен И сервис /oled_mouth/play_audio есть:                    │
│   → вызов сервиса (звук + анимация)                                      │
│ Иначе:                                                                   │
│   → SoX (play) — только звук, без анимации                               │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼ (если через ROS)
┌─────────────────────────────────────────────────────────────────────────┐
│ robot_mouth_talk_node — ОДНОВРЕМЕННО                                     │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. Воспроизводит звук (pygame/ALSA) → звуковая карта → динамики          │
│ 2. Рисует анимацию рта на OLED 0x3D (амплитуда по RMS или синусоиде)     │
│ Старт звука и анимации — в один момент                                  │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3. Что за библиотеки и зачем

| Библиотека | Где используется | Зачем |
|------------|------------------|-------|
| **rospy** | voice_play, robot_mouth_talk_node | ROS — обмен сообщениями между узлами |
| **ainex_interfaces** | voice_play | Схема сервиса SetString (путь к файлу) |
| **pygame** | robot_mouth_talk_node | Воспроизведение MP3/WAV через ALSA |
| **luma.oled** | robot_mouth_talk_node | Рисование на OLED I2C 0x3D |
| **Pillow** | robot_mouth_talk_node | Создание изображений рта |
| **numpy, soundfile, scipy** | robot_mouth_talk_node | Анализ громкости (RMS) для анимации по звуку |

**Установка (если чего-то нет):**
```bash
pip3 install pygame luma.oled pillow
# для анимации по громкости:
pip3 install numpy soundfile scipy
```

## 4. Конфликт: oled_display и robot_mouth_talk_node

Оба используют **один и тот же OLED-дисплей 0x3D** (рот):

| Сервис/нода | Дисплей 0x3C | Дисплей 0x3D |
|-------------|---------------|--------------|
| **oled_display.service** | IP, CPU, батарея | Статичный рот |
| **robot_mouth_talk_node** | — | Анимированный рот + звук |

**Нельзя** запускать оба одновременно — они будут конфликтовать на 0x3D.

**Выбор:**
- **Вариант А:** Используете звук + анимацию → останавливаете `oled_display`, работает `robot_mouth_talk_node`
- **Вариант Б:** Используете инфо (IP, батарея) на 0x3C и статичный рот → работает `oled_display`, отключаете `oled_mouth` в base.launch

## 5. Как найти путь к ROS

Если вы не знаете, где установлен ROS:

```bash
# Проверить, есть ли ROS и где
ls /opt/ros/
# Обычно: noetic, melodic или foxy

# Путь к ROS (если найдено):
# /opt/ros/noetic  — для ROS Noetic
# /opt/ros/melodic — для ROS Melodic

# Путь к workspace (ваш проект):
# /home/ubuntu/ros_ws  — на роботе Ubuntu
# /home/pi/ros_ws      — на Raspberry Pi
```

**Загрузка окружения:**

```bash
# 1. ROS (подставьте noetic или melodic по необходимости)
source /opt/ros/noetic/setup.bash

# 2. Workspace (подставьте свой путь)
source /home/ubuntu/ros_ws/devel/setup.bash
# или, если собирали через colcon:
source /home/ubuntu/ros_ws/install/setup.bash
```

**Автоматически:** скрипт `ainex_bringup/scripts/source_env.bash` ищет ROS и workspace и подгружает их. Его используют systemd-сервисы.

## 6. Пошаговая настройка (звук + анимация)

### Шаг 1. Сборка

```bash
# Перейдите в workspace (укажите свой путь, если отличается)
cd /home/ubuntu/ros_ws
# или: cd /home/pi/ros_ws

source /opt/ros/noetic/setup.bash   # или melodic
catkin_make
# или: colcon build
source devel/setup.bash
# или: source install/setup.bash
```

### Шаг 2. Остановить oled_display

```bash
sudo systemctl stop oled_display.service
# Чтобы не запускался при перезагрузке:
sudo systemctl disable oled_display.service
```

### Шаг 3. Запуск робота (bringup)

Робот обычно стартует через systemd:

```bash
sudo systemctl start start_app_node.service
```

`start_app_node.service` запускает `bringup.launch` → `base.launch` → `oled_mouth.launch` → **robot_mouth_talk_node**.

Либо вручную в терминале:

```bash
# Вариант 1: если знаете путь к ros_ws
cd /home/ubuntu/ros_ws   # или /home/pi/ros_ws
source /opt/ros/noetic/setup.bash
source devel/setup.bash
roslaunch ainex_bringup bringup.launch

# Вариант 2: автоматический поиск ROS (скрипт ищет пути сам)
/home/ubuntu/ros_ws/src/ainex_bringup/scripts/source_env.bash roslaunch ainex_bringup bringup.launch
```

### Шаг 4. Проверка

В **другом терминале** (с `source devel/setup.bash`):

```bash
# Сервис доступен?
rosservice list | grep oled_mouth

# Должно быть:
# /oled_mouth/play_audio
```

Если сервис есть — нода работает.

### Шаг 5. Тест воспроизведения

```bash
rosservice call /oled_mouth/play_audio "data: 'running'"
```

**Ожидаемо:**
- Звук идёт из динамиков
- На OLED 0x3D рот анимируется синхронно со звуком

## 6. Где лежат звуковые файлы

| Имя | Путь |
|-----|------|
| running, warnning | `ainex_sdk/audio/english/running.wav`, `warnning.wav` |
| Свои файлы | `ainex_bringup/voice/` или полный путь |

Примеры:

```bash
rosservice call /oled_mouth/play_audio "data: 'running'"
rosservice call /oled_mouth/play_audio "data: 'warnning'"
rosservice call /oled_mouth/play_audio "data: 'adam-dragon-us_3475244.mp3'"
rosservice call /oled_mouth/play_audio "data: '/home/ubuntu/ros_ws/src/ainex_bringup/voice/my_sound.mp3'"
```

## 7. Как робот «реагирует» на это

1. **При старте приложения (app_node):** вызывается `voice_play.play('running')` → если ROS и сервис работают → звук «running» + анимация рта.

2. **При низкой батарее (oled_display):** вызывается `voice_play.play('warnning')` — но `oled_display` при этом должен быть остановлен, иначе конфликт по дисплею. Предупреждение о батарее в этом режиме не будет показываться через oled_display (можно реализовать отдельно в robot_mouth_talk_node).

3. **Из вашего кода:**
   ```python
   from ainex_sdk import voice_play
   voice_play.play('running', language='English')
   ```
   Если `robot_mouth_talk_node` запущен — звук и анимация пойдут автоматически.

4. **Вручную (команда, скрипт, приложение):**
   ```bash
   rostopic pub /oled_mouth/audio_path std_msgs/String "data: 'running'" --once
   ```
   или
   ```bash
   rosservice call /oled_mouth/play_audio "data: 'running'"
   ```

## 8. Если не работает — проверки

### Нет звука

```bash
# Проверка звука
aplay -l
amixer sget Master
# Тестовый звук
speaker-test -t wav -c 1
```

### Нет анимации

- Убедитесь, что `oled_display.service` остановлен.
- Проверьте, что `robot_mouth_talk_node` запущен: `rosnode list | grep robot_mouth`.

### Сервис не найден

```bash
rosservice list | grep oled_mouth
```

Если пусто — нода не запущена. Проверьте:

- Запущен ли `start_app_node.service` или `roslaunch ainex_bringup bringup.launch`
- В `base.launch` есть `oled_mouth=true` (по умолчанию включён)

### voice_play не идёт через ROS

`voice_play` вызывает ROS только если:

1. `rospy` уже инициализирован (вызов из ROS-ноды, например app_node)
2. Сервис `/oled_mouth/play_audio` существует

Если вызываете `voice_play.play()` из обычного скрипта без ROS — будет использоваться SoX.

---

## 9. Отключение oled_mouth

Если анимация рта не нужна (например, хотите только oled_display):

```bash
roslaunch ainex_bringup bringup.launch oled_mouth:=false
```

Или в `base.launch` изменить `default="true"` на `default="false"` для аргумента `oled_mouth`.

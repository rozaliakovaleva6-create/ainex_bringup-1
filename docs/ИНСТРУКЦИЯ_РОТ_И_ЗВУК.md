# Полная инструкция: имитация разговора на втором дисплее при наличии звука

## Что это

Система воспроизводит звук и **синхронно** показывает анимацию рта на **втором OLED-дисплее** (адрес 0x3D). Пока играет звук — рот открывается и закрывается по громкости; по окончании — остаётся стандартный рот.

**Два дисплея:**
- **0x3C** — инфо (IP, батарея, CPU) — используется `oled_display`
- **0x3D** — рот/эмоции — используется для анимации рта

---

## Два режима работы

### Режим 1: Автономный скрипт (без ROS) — `robot_mouth_talk.py`

Для быстрого теста без ROS. Воспроизводит один файл и рисует рот на дисплее 0x3D.

### Режим 2: ROS-нода — `robot_mouth_talk_node`
Для интеграции с ROS. Аудио приходит из топика или сервиса. Подробнее: [docs/ЗВУК_И_АНИМАЦИЯ_РОТА.md](ЗВУК_И_АНИМАЦИЯ_РОТА.md).

## Пошаговая инструкция: автономный режим (robot_mouth_talk.py)

### Шаг 1. Установка зависимостей

```bash
# Обязательные
pip3 install pygame luma.oled pillow

# Для анимации по громкости (рекомендуется)
pip3 install numpy soundfile scipy

# Для MP3 (опционально, если не работает soundfile)
pip3 install pydub
# pydub требует ffmpeg:
sudo apt install ffmpeg
```

### Шаг 2. Подготовка аудиофайла

Положите аудиофайл в одну из папок (по приоритету):

1. **`ainex_bringup/voice/`** — основная папка
2. Папка SDK: `ainex_sdk/audio/`
3. `/home/pi/Music/` или `/home/ubuntu/Music/`

Или задайте полный путь в переменной окружения:

```bash
export ROBOT_MOUTH_AUDIO=/полный/путь/к/файлу.mp3
```

**Поддерживаемые форматы:** WAV, MP3 (через pygame/mpv/ffplay для воспроизведения; для анимации по громкости — WAV или MP3 с pydub).

### Шаг 3. Смена файла (если нужно)

В `scripts/robot_mouth_talk.py` измените строку:

```python
AUDIO_FILENAME = "adam-dragon-us_3475244.mp3"  # ← ваше имя файла
```

### Шаг 4. Остановка конфликтующих сервисов

Дисплей 0x3D может использоваться только одним процессом. Остановите:

```bash
sudo systemctl stop oled_display.service
# Если используется motik (emotions):
# отключите emotions_display в bringup.launch или остановите соответствующий сервис
```

### Шаг 5. Проверка I2C и дисплея

```bash
# Сканирование I2C-шины
sudo i2cdetect -y 1
# Должны быть 0x3C и 0x3D
```

### Шаг 6. Запуск

```bash
cd /home/ubuntu/ros_ws
source /opt/ros/noetic/setup.bash   # или melodic
source devel/setup.bash

# Запуск
rosrun ainex_bringup robot_mouth_talk.py
```

Или напрямую (без ROS):

```bash
cd /home/ubuntu/ros_ws/src/ainex_bringup/scripts
python3 robot_mouth_talk.py
```

**Ожидаемо:**
- Звук идёт из динамиков
- На дисплее 0x3D рот анимируется синхронно со звуком
- По окончании — остаётся стандартный рот.

**Выход:** Ctrl+C

## Устранение неполадок

### Нет звука

```bash
# Проверка устройств
aplay -l
amixer sget Master

# Тест
aplay /usr/share/sounds/alsa/Front_Center.wav
```

Если нужен другой выход (USB-карта):
```bash
ROBOT_MOUTH_AUDIO=/path/file.mp3 AUDIODEV=plughw:2,0 python3 robot_mouth_talk.py
```

### Файл не найден

Скрипт выведет подсказку. Проверьте:
- Файл в `ainex_bringup/voice/`
- Имя совпадает с `AUDIO_FILENAME`
- Или переменная `ROBOT_MOUTH_AUDIO`

### Дисплей не отвечает

```bash
# Ошибка "Дисплей I2C 0x3D: ..."
# 1. Проверьте I2C
sudo i2cdetect -y 1

# 2. Убедитесь, что дисплей не занят
sudo systemctl stop oled_display.service
ps aux | grep -E "oled|robot_mouth"
```

### Рот не двигается по громкости (только синусоида)

Установите `numpy`, `soundfile`, `scipy`. Для MP3 — `pydub` и `ffmpeg`.

### Ошибка pygame

```bash
# На headless (без монитора)
export SDL_VIDEODRIVER=dummy
export SDL_AUDIODRIVER=alsa
```

| Файл | Назначение |
|------|------------|
| `robot_mouth_talk.py` | Автономный скрипт: звук + анимация на 0x3D |

## ROS-режим (кратко)

```bash
# Остановить oled_display
sudo systemctl stop oled_display.service

# Запуск
roslaunch ainex_bringup oled_mouth.launch

# В другом терминале — воспроизведение
rosservice call /oled_mouth/play_audio "data: 'running'"
rostopic pub /oled_mouth/audio_path std_msgs/String "data: 'имя_файла'" --once
```

**Важно:** В текущей конфигурации `robot_mouth_talk_node` может не выводить на дисплей 0x3D (вывод отключён, если дисплей используется motik). Для вывода рта на 0x3D используйте автономный `robot_mouth_talk.py` или настройте конфликт с motik.
